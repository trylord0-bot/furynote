## 2. 기술 스택

| 항목 | 기술 | 비고 |
|---|---|---|
| **프론트엔드** | Flutter | iOS / Android 하이브리드 |
| **백엔드** | Python — **FastAPI** | |
| **ORM** | SQLAlchemy | MariaDB 연동 |
| **DB** | MariaDB | |
| **STT** | 기기 내장 STT | `speech_to_text` 패키지 / iOS → Apple Speech, Android → Google Speech |
| **모더레이션** | OpenAI Moderation API | 포스팅 필터링 전용 |
| **기기 식별** | Device ID 기반 | 회원가입 없이 어뷰징 추적 |
| **푸시 알림 (서버)** | FCM (Firebase Cloud Messaging) | `firebase-admin` SDK / Python 백엔드에서 댓글 알림 발송 |
| **푸시 알림 (로컬)** | flutter_local_notifications | 리마인드 알림 / 주간 리포트 알림 |
| **인앱 결제** | in_app_purchase (Flutter) | iOS App Store / Google Play 통합 처리 |
| **영수증 검증** | httpx | Python 백엔드에서 App Store / Google Play API 호출 |
| **백업 암호화** | AES-256 + 앱 내장 고정 키 | `--dart-define`으로 빌드 타임 주입 |

### 백엔드 주요 패키지
```
FastAPI
├── SQLAlchemy        ← MariaDB 연동
├── firebase-admin    ← FCM 푸시 발송
├── openai            ← Moderation API
└── httpx             ← App Store / Google Play 영수증 검증
```

### STT 상세
- Flutter 패키지 `speech_to_text`를 통해 **기기 내장 STT 엔진** 사용
- iOS: Apple Speech Framework / Android: Google Speech Recognition
- **한국어 지원 ✅**
- 별도 서버 처리 없음, 기기에서 직접 처리 후 텍스트로 변환하여 저장
- 내장 STT 미지원 기기에서는 음성 입력 버튼 비활성화 처리

## 6. 노트(기록) 시스템

### 6-1. 노트 작성 흐름 — 6스텝 구성

> 각 스텝은 **하나의 풀스크린 화면**으로 구성되며,
> 선택 완료 시 자동으로 다음 스텝으로 **아래로 스크롤**되어 이동한다.
> 각 스텝 상단 중앙에는 **↑ 원형 버튼**이 있어 이전 스텝으로 돌아갈 수 있다.
> (Step 1에서는 뒤로가기 버튼 숨김)

| Step | 항목 | 필수 여부 | 설명 |
|:---:|---|:---:|---|
| 1 | **분노 강도** | ✅ 필수 | 1~5단계 이모지 중 1개 선택 → 선택 즉시 다음 스텝으로 자동 스크롤 |
| 2 | **카테고리** | ✅ 필수 | 원인 카테고리 1개 선택 → 선택 즉시 다음 스텝으로 자동 스크롤 |
| 3 | **텍스트 / 음성 입력** | ⬜ 선택 | 자유 서술 또는 음성 입력 / `다음 →` 또는 `건너뛰기` 버튼으로 진행 |
| 4 | **리마인드 설정** | ⬜ 선택 | 토글 ON 후 시간 선택 / `다음 →` 또는 `건너뛰기` 버튼으로 진행 |
| 5 | **저장** | ✅ 필수 | 입력 내용 요약 카드 표시 후 `🔥 기록하기` 버튼으로 저장 |
| 6 | **포스팅** | ⬜ 선택 | 저장 직후 포스팅 카드 미리보기 표시 / `📮 포스팅하기` 또는 `그냥 저장만 할게요` 선택 |

### 6-2. 스텝 UX 상세
- **상단 인디케이터:** 6개의 점으로 현재 스텝 위치 표시
  - 완료된 스텝: 빨간 점 (흐릿)
  - 현재 스텝: 빨간 긴 바
  - 미진행 스텝: 어두운 점
- **배경 틴트:** Step 1에서 분노 강도 선택 시 해당 단계 컬러로 배경이 은은하게 물듦
- **완료된 스텝:** 위에 흐릿하게 남아 흐름 확인 가능
- **뒤로가기:** 각 스텝 상단 중앙의 **↑ 원형 버튼** (Step 1 제외)

### 6-3. 음성 입력 (Step 3)
- 텍스트 입력 대신 음성 입력 선택 가능
- 기기 내장 STT를 통해 음성 → 텍스트 변환 후 저장
- 녹음 중 펄스 애니메이션 표시
- 기기가 STT를 지원하지 않을 경우 음성 입력 버튼 비활성화

### 6-4. 저장 요약 카드 (Step 5)

| 항목 | 내용 |
|---|---|
| 분노 강도 | 이모지 + Lv.N + 한국어 레이블 |
| 카테고리 | 이모지 + 카테고리명 |
| 내용 | 입력한 텍스트 (없으면 '없음') |
| 리마인드 | 설정한 시간 (없으면 '없음') |

### 6-5. 포스팅 카드 미리보기 (Step 6)
- Step 5에서 `🔥 기록하기` 버튼 탭 시 Step 6으로 자동 이동
- 실제 입력 내용이 반영된 **포스팅 카드 미리보기** 표시
- 카드 구성: 분노 강도 이모지 + 닉네임 + Lv. 뱃지 + 텍스트 + 카테고리 + 시간
- `📮 포스팅하기` → 피드에 공개 게시
- `그냥 저장만 할게요` → 포스팅 없이 개인 기록으로만 저장

### 6-6. 포스팅(공유) 데이터 구조

```json
{
  "device_id": "기기 고유 ID",
  "nickname": "화난 호랑이#4827",
  "rage_level": 3,
  "category": "driving",
  "text": "옆 차선 차가 갑자기 끼어들었는데 사과도 없이 가버림."
}
```

## 7. 리마인드 & 재기록 시스템

### 7-1. 개요
- 노트 작성 시 사용자가 **리마인드 시간을 직접 설정**할 수 있다.
- 설정한 시간이 되면 푸시 알림으로 해당 노트를 다시 보여준다.
- 핵심 목적: **"그때 그 분노, 지금은 얼마나 사그라졌나요?"** 를 스스로 돌아보게 하는 기능

### 7-2. 리마인드 시간 선택지

| 선택지 | 비고 |
|---|---|
| 30분 후 | |
| 1시간 후 | |
| 2시간 후 | |
| 6시간 후 | |
| 내일 | |
| 직접 설정 | 사용자가 직접 시간 입력 |

### 7-3. 동작 흐름

```
노트 작성 시 리마인드 시간 설정 (선택)
      ↓
설정한 시간 도달 시 푸시 알림 발송
예) "🔥 아까 그 분노, 지금은 어때요?"
      ↓
알림 탭 시 → 진정 탭의 리마인드 노트 목록으로 이동
      ↓
리마인드 카드 탭 → 바텀 시트 모달 표시
      ↓
현재 진정 정도 선택 (6단계)
🤬 아직 분노 / 😡 많이 화남 / 😠 조금 화남
😤 약간 불편 / 😒 거의 괜찮 / 😌 진정됨 ✓
      ↓
[분기 A] 😌 진정됨 선택 시
      → "진정됨으로 저장 & 삭제" → 리마인드 목록에서 제거
      → 완화 결과 저장 및 인사이트 문구 표시
[분기 B] 분노 단계 선택 시 (1~5단계)
      → "현재 상태 저장" → 리마인드 목록에 유지
      → 강도 2 이상이면 진정 도구 추천 화면으로 연결
      ↓
최초 기록 강도 vs 리마인드 후 강도 비교 저장
```

### 7-4. 리마인드 후 데이터 구조

```json
{
  "note_id": "노트 고유 ID",
  "original_rage_level": 4,
  "reminded_at": "2026-06-09T15:30:00",
  "reminded_rage_level": 2
}
```

### 7-5. 정책
- 리마인드 시간은 노트 작성 시 또는 작성 후 수정 가능
- **리마인드는 노트 1개당 1회만 설정 가능**
- 리마인드 후 강도 재입력은 선택 사항 (무시해도 무방)
- **리마인드 알림 방식: 로컬 푸시** (`flutter_local_notifications` 패키지 사용)
- 리마인드 목록은 **진정 탭 상단**에 표시
- 전체 진정됨 처리 시 빈 상태 메시지 표시: "😌 모두 진정됐군요! 잘 하셨어요 🎉"

## 8. 진정 도구 시스템

### 8-1. 개요
- 리마인드 재기록 후 **아직 화가 남아있을 때** 자동으로 추천되는 감정 완화 도구 모음
- 바텀 네비 **진정 탭**에서도 독립적으로 접근 가능

### 8-2. 도구 목록

| 도구 | 포인트 컬러 | 소요 시간 | 설명 |
|---|---|---|---|
| **호흡운동** | `#7EC8E3` (하늘) | 3~5분 | 4-7-8 호흡법으로 심박수 낮추기 |
| **타임아웃** | `#A8D8A8` (민트) | 10분 | 공간 이동, 물리적 분리 |
| **이미지 명상** | `#C9B8FF` (보라) | 5~10분 | 평온한 장면 시각화 |
| **포스팅으로 위로받기** | `#FFB347` (주황) | — | 피드 탭으로 이동, 비슷한 분노 공감 |

### 8-3. 각 도구 상세 가이드

| 도구 | 가이드 내용 |
|---|---|
| 호흡운동 | ① 코로 4초 들이마시기 → ② 7초 숨 참기 → ③ 입으로 8초 내쉬기 / 3~4회 반복 |
| 타임아웃 | ① 현재 공간에서 물리적으로 자리 피하기 → ② 조용한 곳 10분 혼자 있기 → ③ 스마트폰 내려놓기 |
| 이미지 명상 | ① 눈 감고 편하게 앉기 → ② 바다/숲/좋아하는 공간 생생하게 상상 → ③ 소리·냄새·온도 느끼기 |
| 포스팅으로 위로받기 | 피드에서 비슷한 상황 포스팅 탐색 또는 내 이야기 공유 |

### 8-4. 정책
- 각 도구 탭 시 바텀 시트로 상세 가이드 표시
- 리마인드 재기록 시 분노 강도 2 이상이면 자동 추천
- 사용자가 직접 선택하여 사용도 가능

## 9. 커뮤니티 (포스팅) 시스템

### 9-1. 개요
- 작성한 노트를 기반으로 **자동 카드 형태**로 포스팅 생성
- 기본값은 **익명 공개**
- 비슷한 분노를 가진 사람들과 공감을 나누는 공간

### 9-2. 포스팅 카드 구성
- 분노 강도 이모지 + Lv. 뱃지
- 닉네임 (익명)
- 텍스트 (있을 경우)
- 카테고리
- 작성 시간
- 공감 숫자 카운트 + 댓글 수

### 9-3. 반응 시스템
- **공감:** 포스팅에 공감 버튼 탭 → 숫자 카운트 증가
- **댓글:** 포스팅에 댓글 작성 가능

### 9-4. 피드 정렬
- **최신순 단일 정렬**

### 9-5. 콘텐츠 필터링
> ⚠️ 필터링은 포스팅(공개 공유) 시에만 적용. 개인 노트 저장에는 미적용.
> 텍스트 없는 포스팅은 Step 1~3 생략, Step 4(어뷰징 감지)만 적용.

```
포스팅 등록 API 요청
      ↓
[텍스트 있을 경우]
[Step 1] URL 포함 여부 → 차단
[Step 2] 광고성 키워드 필터 → 차단
[Step 3] OpenAI Moderation API → 차단
      ↓
[항상 실행]
[Step 4] 단시간 다량 포스팅 감지
         동일 device_id 기준 60초 내 5회 이상 → 차단
      ↓
✅ 통과 → DB 저장 → 피드 노출
❌ 차단 → 거절 응답 (사유 코드 포함)
```

### 9-6. 어뷰징 감지 정책

| 항목 | 내용 |
|---|---|
| **감지 기준** | 동일 `device_id` 기준 **60초 내 5회 이상** 포스팅 시도 |
| **처리 방식** | 해당 요청 차단 (일시 차단, DB 저장 안 됨) |
| **차단 해제** | 자동 해제 (시간 경과 후 자연 해소) |
| **사유 코드** | `RATE_LIMIT_EXCEEDED` |

### 10-1. 개요

| **분노 감소율** | 리마인드 전후 강도 비교 기반 평균 감소 수치 |

| 요소 | 상세 |

| **분노 강도 추이** | 날짜별 분노 강도 변화 라인 그래프 / Y축: 이모지 5단계 / 오늘 포인트 강조 |

| **원인별 분포** | 도넛 파이 차트 + 범례 / 도넛 중앙에 1위 카테고리 % + 이름 표시 |
| **캘린더 히트맵** | 날짜별 분노 강도를 색상으로 표시 |
| **개인 인사이트 문구** | 통계 기반 자동 생성 (예: "이번 주 직장에서 가장 많이 화났어요") |

## 12. 알림 설정 시스템

### 12-1. 알림 종류 및 정책

| 알림 종류 | 방식 | 기본값 | 설명 |
|---|---|:---:|---|
| **리마인드 알림** | 로컬 푸시 | ON | 노트 작성 시 설정한 시간에 발송 |
| **댓글 알림** | 서버 푸시 (FCM) | ON | 내 포스팅에 댓글이 달릴 때 발송 |
| **주간 리포트 알림** | 로컬 푸시 | ON | 매주 월요일, 지난 주 통계 요약 발송 |

### 12-2. 기술 상세

#### 로컬 푸시 (리마인드 알림 / 주간 리포트 알림)
- Flutter 패키지: `flutter_local_notifications`
- **리마인드 알림:** 노트 저장 시 설정된 시간으로 로컬 알림 스케줄 등록
- **주간 리포트 알림:** 앱 실행 시 또는 매주 월요일 자정 기준으로 다음 월요일 알림 재등록
- 알림 탭 시 해당 화면으로 딥링크 이동

#### 서버 푸시 (댓글 알림)
- **발송 주체:** FastAPI 백엔드
- **발송 수단:** FCM (Firebase Cloud Messaging)
- **FCM 토큰 관리:**
  - 앱 최초 실행 시 FCM 토큰 발급 → 백엔드 등록 API 호출
  - 토큰 갱신 시 자동 업데이트
- **발송 트리거:** 댓글 생성 API 처리 완료 후 포스팅 작성자의 FCM 토큰으로 푸시 발송
- **FastAPI 백엔드 처리 흐름:**
  ```
  댓글 등록 API 요청
        ↓
  DB에 댓글 저장
        ↓
  포스팅 작성자의 device_id로 FCM 토큰 조회
        ↓
  FCM 토큰 존재 + 댓글 알림 ON 상태 확인
        ↓
  firebase-admin SDK로 푸시 발송
  예) "💬 화난 호랑이#1234님이 댓글을 남겼어요"
  ```

### 12-3. 알림 설정 저장
- 각 알림 ON/OFF 상태는 **기기 로컬(SharedPreferences)** 에 저장
- 댓글 알림 OFF 시 백엔드에도 상태 전달 → 불필요한 FCM 발송 방지

### 12-4. DB 스키마 (알림 관련)

```sql
CREATE TABLE device_tokens (
  id             BIGINT AUTO_INCREMENT PRIMARY KEY,
  device_id      VARCHAR(128) NOT NULL UNIQUE,
  fcm_token      VARCHAR(256) NOT NULL,
  notify_comment TINYINT(1) DEFAULT 1 COMMENT '댓글 알림 ON/OFF',
  updated_at     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 14. 데이터 백업 & 복원 시스템

### 14-1. 개요
- PRO 구매 사용자에게 제공되는 **전체 데이터 내보내기 / 가져오기** 기능
- 단일 `.fnbackup` 파일 하나로 모든 데이터를 내보내고 복원

### 14-2. 내보내기 범위

| 포함 ✅ | 미포함 ❌ |
|---|---|
| 전체 노트 기록 (기간 무관) | 커뮤니티 포스팅 |
| 리마인드 재기록 데이터 | 포스팅 댓글 |
| 통계 / 챌린지 달성 기록 | 포스팅 공감 반응 |
| 프로필 사진 (이미지 파일) | |
| **닉네임 + 고유번호** (`화난 호랑이#4827` 형식) | |
| 알림 설정값 | |

### 14-3. 파일 구조

`.fnbackup` 파일의 실체는 **AES-256으로 암호화된 ZIP 아카이브**

```
furynote_backup_YYYYMMDD_HHmmss.fnbackup
└── (AES-256 암호화 ZIP)
    ├── manifest.json         ← 버전 정보, 생성일시, 체크섬
    ├── data.json             ← 전체 데이터 (노트, 리마인드, 설정, 프로필 등)
    └── assets/
        ├── avatar.jpg        ← 프로필 사진 (설정한 경우)
        └── (추후 확장 가능)
```

#### manifest.json 구조
```json
{
  "version": "1.0",
  "app_version": "1.0.0",
  "created_at": "2026-06-10T21:00:00+09:00",
  "device_id": "기기 고유 ID",
  "checksum": "sha256_hash_of_data_json"
}
```

#### data.json 구조
```json
{
  "profile": {
    "display_name": "화난 호랑이#4827",
    "nickname": "화난 호랑이",
    "unique_number": "4827",
    "avatar": "assets/avatar.jpg"
  },
  "settings": {
    "notify_remind": true,
    "notify_comment": true,
    "notify_weekly": true
  },
  "notes": [ ... ],
  "reminders": [ ... ],
  "challenges": [ ... ]
}
```

### 14-4. 암호화 방식

| 항목 | 내용 |
|---|---|
| **알고리즘** | AES-256-CBC |
| **키 방식** | 앱 내장 고정 키 (`--dart-define`으로 빌드 타임 주입) |
| **무결성 검증** | manifest.json 내 SHA-256 체크섬으로 위변조 감지 |
| **Flutter 패키지** | `encrypt` 패키지 사용 |

### 14-5. 내보내기 흐름

```
설정 → 데이터 내보내기 → PRO 확인
      ↓
전체 데이터 수집
(노트, 리마인드, 설정, 닉네임#고유번호, 프로필 사진)
      ↓
data.json 생성 + assets/ 이미지 수집
      ↓
manifest.json 생성 (SHA-256 체크섬 포함)
      ↓
ZIP 압축 → AES-256 암호화
      ↓
.fnbackup 파일 생성
      ↓
기기 공유 시트(Share Sheet)로 저장 위치 선택
예) iCloud / Google Drive / 로컬 저장 등
```

### 14-6. 가져오기(복원) 흐름

```
설정 → 데이터 가져오기 → .fnbackup 파일 선택
      ↓
AES-256 복호화
      ↓
manifest.json 파싱 → 버전 호환성 확인
      ↓
SHA-256 체크섬 검증 → 파일 위변조 확인
      ↓
data.json 파싱 → 기존 데이터와 병합
(중복 note_id는 자동 제거)
      ↓
닉네임#고유번호 복원
→ 백업 파일의 nickname / unique_number / display_name 으로
  현재 기기의 닉네임#고유번호를 묻지 않고 덮어쓰기
      ↓
assets/ 이미지 복원 (프로필 사진 등)
      ↓
✅ 복원 완료 토스트 표시
❌ 실패 시 오류 메시지 표시 (파일 손상 / 버전 불일치 등)
```

### 14-7. 정책
- 내보내기는 **PRO 구매자만** 사용 가능
- 가져오기(복원)는 **무료 사용자도** 사용 가능
- 복원 시 기존 데이터에 **병합** 처리 (덮어쓰기 아님)
- 중복 `note_id` 는 자동 제거
- **닉네임#고유번호는 백업 파일 기준으로 무조건 덮어쓰기** (팝업 없음)
- 버전 불일치 시 경고 후 진행 여부 사용자에게 확인
